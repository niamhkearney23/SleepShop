1. What this system does

When a candidate scans the QR code (or visits the link), they complete a 20-minute AI-scored assessment in their browser. Behind the scenes:

  • Their resume is uploaded to the firm's Google Drive folder
  • Their answers are scored by AI (Claude) across 10 qualities out of 100
  • A formatted Google Doc report is auto-generated and saved
  • A row is added to the master Google Sheet with a link to the report
  • 24 hours later, an automatic email is sent: Shortlist (with Calendly link), Under Review, or Polite Decline

Your only manual steps: (a) review applications within the 24-hour window and override anything the AI got wrong, and (b) send the formal acceptance letter to shortlisted candidates after the screening call.

3. The applicant flow

  1. 1Day 0 — Candidate scans QR code or visits the link. Completes 20-minute assessment.
  2. 2Day 0 — They see a "Thank You" screen (no scores shown to them).
  3. 3Day 0 — Behind the scenes: data saves to Sheet, resume to Drive, AI scores all 10 qualities, Google Doc report generates. Email status set to SCHEDULED for 24h later.
  4. 4Within 24h — You review in the Sheet. Override or cancel any email you disagree with.
  5. 5Day 1 — Auto-email sends at the scheduled time. Shortlist = Calendly link. Borderline = "under review". No = polite decline.
  6. 6Day 1–3 — Shortlisted candidate clicks Calendly link, books 15-min screening call.
  7. 7Day 2–5 — Screening call happens via Google Meet (auto-generated by Calendly). Niamh attends.
  8. 8After the call — Niamh opens the Sheet, finds the row, picks ACCEPT or REJECT from the Post-Interview dropdown (column R). Diviya instantly receives an email with the candidate's details + the Report link + which letter to send.
  9. 9Diviya — reads the email, sends the formal acceptance letter (with 2-week trial details) or a polite decline, then ticks the "Letter Sent" checkbox (column T) on the Sheet. The "Sent On" timestamp auto-fills so Niamh can see at a glance that the loop is closed.

6. Overriding an AI decision

You have a 24-hour window to override anything before the auto-email goes out. After that, the email has already sent and you'd need to email the candidate manually.

To change a decision (e.g. No → Yes)

  1. Open the Sheet, find the candidate's row
  2. Click cell in Decision column (L)
  3. Change NO to YES
  4. Leave Email Status as SCHEDULED
  5. The next hourly trigger (max 1 hour wait) will send the right email

To cancel an email entirely

  1. Find the candidate's row
  2. Change Email Status from SCHEDULED to CANCELLED
  3. No email will ever be sent for this row

To send immediately (skip the 24-hour wait)

  1. Change Send At to a time in the past (e.g. yesterday)
  2. Wait up to 1 hour for the next trigger
  3. Or, to send within 30 seconds: open Apps Script → select function sendAllDueNow → click Run

7. Updating the Calendly link

The shortlist email currently says "a member of our team will be in touch" because Calendly hasn't been approved yet. Once it is:

  1. Get the Calendly event link (looks like https://calendly.com/firm/paralegal-screening)
  2. Open Apps Script → Code.gs
  3. Find the line: var CALENDLY_LINK = 'https://calendly.com/YOUR-LINK-HERE';
  4. Replace with the real URL
  5. Save (Cmd + S)
  6. Click Deploy → Manage deployments → pencil icon ✏️ → Version: New version → Deploy

From that moment on, every shortlist email automatically includes the Calendly link.

8. Exporting TPAS CRM subscribers

Every Monday 9am, a CSV of candidates who opted into the Thomas Philip CRM is emailed automatically to nmk@lawgistics.my. Forward this file to Diviya, who imports it into the Thomas Philip CRM.

To export manually at any time

  1. Open Apps Script
  2. In the function dropdown (next to the Run button), pick exportTPASSubscribersCSV
  3. Click Run ▶️
  4. The CSV is saved to the Drive folder and emailed to nmk@lawgistics.my
  5. Forward to Diviya for CRM import

CSV format: email, first_name, last_name — standard format that works with most CRM tools.

9. Troubleshooting

A submission isn't showing up in the Sheet

  • Open Apps Script → Executions tab (4th icon on left sidebar)
  • Look for a recent doPost row
  • If it's green / Completed but nothing in Sheet → check tab name is exactly Applications
  • If it's red / Failed → click the row to see the error message
  • If there's no doPost at all → the Netlify site isn't reaching the Apps Script. Re-deploy Apps Script as "Web app" with access set to Anyone

Auto-emails stopped sending

  • Open Apps Script → Triggers tab (clock icon)
  • Verify processPendingEmails is listed and running every hour
  • If missing: run setupAllTriggers once from the editor to recreate all triggers

AI scoring returns "ERROR"

  • Means the Anthropic API call failed — usually an API key issue or credit limit
  • Check console.anthropic.com for credits/billing
  • API key lives in the assessment HTML (not the Apps Script)

Candidate reports they didn't get an email

  • Ask them to check spam
  • Check the Sheet — is Email Status SENT or still SCHEDULED?
  • If SCHEDULED → hasn't sent yet (within the 24h delay)
  • If SENT → it went out — likely in their spam folder
